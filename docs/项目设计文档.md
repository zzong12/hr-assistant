# HR数字助手 - 项目设计文档

## 📋 项目概述

### 项目简介
**HR数字助手** 是一个基于人工智能的招聘管理系统，旨在通过多Agent架构和现代化的Web界面，帮助HR专业人士高效完成职位管理、简历筛选、面试协调和候选人沟通等核心招聘流程。

### 项目目标
- 🎯 **提升招聘效率**: 通过AI自动化重复性工作，减少50%的手工操作时间
- 🎯 **提高决策质量**: 基于数据分析和智能匹配，提供更精准的招聘决策支持
- 🎯 **优化候选人体验**: 快速响应、专业沟通，提升雇主品牌形象
- 🎯 **降低招聘成本**: 减少不合适的面试安排，优化资源配置

### 核心价值
| 价值维度 | 传统方式 | HR数字助手 | 提升幅度 |
|---------|---------|-----------|---------|
| JD撰写时间 | 30-60分钟 | 5-10分钟 | ⬇️ 80% |
| 简历筛选时间 | 每天2-4小时 | 每天30分钟 | ⬇️ 75% |
| 面试准备时间 | 15-30分钟/场 | 5分钟/场 | ⬇️ 70% |
| 候选人响应速度 | 1-3天 | 实时 | ⬆️ 95% |
| 匹配准确率 | 60-70% | 85-95% | ⬆️ 35% |

### Agent详细设计

#### 1️⃣ HR Concierge (主控Agent)

**角色定位:** 统一入口、任务理解与分发、结果整合

**核心职责:**
- 🎯 理解HR用户的需求和意图
- 🎯 识别任务类型并分发给合适的专家Agent
- 🎯 整合专家Agent的输出，提供清晰友好的回复
- 🎯 维护对话上下文，理解指代关系
- 🎯 协调多个Agent完成复杂任务

**工作流程:**
```
1. 接收用户输入
   ↓
2. 分析意图和上下文
   ↓
3. 判断任务类型
   ├─ → JD生成 → JD生成专员
   ├─ → 简历处理 → 简历筛选专员
   ├─ → 面试安排 → 面试协调专员
   ├─ → 沟通话术 → 候选人沟通专员
   └─ → 其他 → 直接处理或询问
   ↓
4. 整合专家输出
   ↓
5. 生成友好回复
   ↓
6. 询问是否需要进一步帮助
```

**系统提示词框架:**

#### 3️⃣ 简历筛选专员 (Resume Specialist)

**角色定位:** 简历评估与匹配专家

**核心能力:**
- 🔍 解析简历，提取关键信息
- 🔍 根据职位JD计算匹配度分数(0-100)
- 🔍 提供明确的推荐理由和面试建议
- 🔍 识别候选人的亮点和潜在风险

**匹配度评估模型:**
```typescript
interface MatchAssessment {
  score: number;                 // 总分 (0-100)
  breakdown: {
    skills: {                    // 技能匹配 (40%)
      score: number;             // 0-100
      matched: string[];         // 匹配的技能
      missing: string[];         // 缺失的技能
    };
    experience: {                // 经验相关性 (30%)
      score: number;             // 0-100
      relevant_years: number;    // 相关工作年限
      industry_match: boolean;   // 行业匹配
    };
    education: {                 // 教育背景 (15%)
      score: number;             // 0-100
      degree_match: boolean;     // 学位匹配
      major_relevance: number;   // 专业相关性 (0-100)
    };
    projects: {                  // 项目经历 (15%)
      score: number;             // 0-100
      relevant_count: number;    // 相关项目数量
      quality: string;           // high/medium/low
    };
  };
  recommendation: {
    decision: "proceed" | "hold" | "reject";
    reason: string;              // 决策理由
    next_steps: string[];        // 建议的下一步行动
  };
  highlights: string[];          // 候选人亮点 (2-3条)
  concerns: string[];            // 潜在问题/风险点
}
```

**系统提示词:**
```typescript
const RESUME_SPECIALIST_PROMPT = `你是一个专业的简历筛选专家，拥有10年招聘和人才评估经验。

你的能力:
1. 解析简历，准确提取关键信息(姓名、联系方式、工作经历、技能、教育背景)
2. 根据职位JD精确计算匹配度分数(0-100)
3. 提供明确、有依据的推荐理由
4. 识别候选人的亮点和潜在问题

匹配度评估标准:
- 技能匹配度 (40%)：评估候选人技能与职位要求的匹配程度
- 经验相关性 (30%)：评估工作经验与职位的相关性
- 教育背景 (15%)：评估学历和专业相关性
- 项目经历 (15%)：评估项目经验的质量和相关性

推荐决策标准:
- 85-100分：强烈推荐，立即安排面试
- 70-84分：推荐，可安排面试
- 60-69分：待定，需要进一步评估
- 60分以下：不推荐

输出格式: 结构化的JSON格式，包含分数、分解、推荐理由

语气:客观、专业、有数据支撑`;
```

```typescript
function calculateMatchScore(resume: Resume, job: Job): number {
  const weights = {
    skills: 0.4,
    experience: 0.3,
    education: 0.15,
    projects: 0.15
  };

  const skillsScore = compareSkills(resume.skills, job.requirements.skills);
  const experienceScore = assessExperience(resume.experience, job.level);
  const educationScore = evaluateEducation(resume.education, job.requirements.education);
  const projectsScore = analyzeProjects(resume.projects, job.description);

  return (
    skillsScore * weights.skills +
    experienceScore * weights.experience +
    educationScore * weights.education +
    projectsScore * weights.projects
  ) * 100;
}
```

## 🚀 部署指南

### 环境要求

#### 开发环境
- Node.js >= 18.17.0
- npm >= 9.0.0
- 现代浏览器（Chrome, Firefox, Safari, Edge）

#### 生产环境
- Node.js >= 18.17.0
- 2GB RAM（推荐4GB）
- 10GB磁盘空间

### 快速开始

```bash
# 1. 克隆项目
git clone <repository-url>
cd hr-assistant

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.local.example .env.local
# 编辑 .env.local，添加 ANTHROPIC_API_KEY

# 4. 启动开发服务器
npm run dev

# 5. 访问应用
# 浏览器打开 http://localhost:3000
```

### 环境变量配置

```env
# .env.local
ANTHROPIC_API_KEY=your_api_key_here

# 可选配置
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_APP_NAME=HR数字助手
```

### 生产部署

#### 方式1: 直接部署
```bash
# 构建
npm run build

# 启动
npm start
```

#### 方式2: 使用PM2
```bash
# 安装PM2
npm install -g pm2

# 启动
pm2 start npm --name "hr-assistant" -- start

# 保存配置
pm2 save

# 设置开机自启
pm2 startup
```

#### 方式3: Docker部署
```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]
```

```bash
# 构建镜像
docker build -t hr-assistant .

# 运行容器
docker run -d -p 3000:3000 --name hr-assistant hr-assistant
```
